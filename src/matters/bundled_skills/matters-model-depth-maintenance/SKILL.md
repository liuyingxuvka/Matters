# Matters Model Depth Maintenance

Read `references/service-contract.md`. Submit every assessment and missing-work
request through MatterService.

1. Bind the assessment to one occurrence, inventory revision, policy,
   capability contract, private execution receipt, prompt/schema, active
   skill, and dependency identity.
2. Check terminal deterministic hard exclusion or admitted coverage, current
   extraction, low-cost annotation, Matter modeling, anchored evidence,
   original-owner terminal disposition, parent-child/WorkItem hierarchy
   audit, bilingual title/summary/topic type, localization, representative
   visual, projection, and UI reachability where required by policy.
3. Return `sufficient` only when every required criterion is satisfied or explicitly non-applicable and no blocker remains.
4. Return `partial` when useful current analysis exists but criteria remain.
5. Return `blocked` for an unavailable required source, extractor,
   ResearchGuard operation, original-owner dispatch, localization, visual, or
   projection stage.
6. Return `stale` whenever a declared dependency identity changes.
7. Queue the exact missing owner work without editing product code,
   requirements, FlowGuard models, or another owner's state.

Ordinary uncertainty is allowed when the responsible owner has recorded a
terminal uncertainty-preserved disposition. It is not a human-review
requirement.

## Shared Matters product contract

The seven model-independent capability roles are `deterministic_preprocessor`,
`low_cost_annotator`, `ambiguity_resolver`, `matter_modeler`,
`hero_image_generator`, `consistency_reviewer`, and
`maintenance_orchestrator`. The preprocessor and annotator cannot create a Matter;
the ambiguity resolver cannot write canonical state. Only `matter_modeler` may
propose parent-child Matters, WorkItems, and bilingual English/Chinese title,
summary, and topic type; `consistency_reviewer` checks but does not author that
state, and the original owner emits the hierarchy audit.

Preserve private `source_neighborhood_id`, `source_group_chain`,
`source_group_labels`, `source_spatial_context_revision`, `path_depth`, and
`file_kind`; require `deterministic_hard_exclusion` before AI. Never expose
absolute paths or private spatial context publicly. Never bind a role to a
named model or price tier, request an API key, call a provider API directly, or
use a direct API fallback.

`hero_image_generator` is presentation-only and `research_operation`/
ResearchGuard is an advisory research lane. Neither may write canonical Matter
state or evidence.

Shared handoff vocabulary: `deterministic_hard_exclusion`,
`deterministic_preprocessor`, `low_cost_annotator`, `ambiguity_resolver`,
`matter_modeler`, `hero_image_generator`, `consistency_reviewer`,
`maintenance_orchestrator`, `source_neighborhood_id`, `source_group_chain`,
`source_group_labels`, `source_spatial_context_revision`, parent, child,
WorkItem hierarchy audit, English/Chinese title, summary, topic type,
`generated_hero`, and `research_operation`. Canonical state never exposes an
absolute path publicly and never binds a named model, API key, direct API, or
fallback.

Treat `generated_hero_terminal` as a presentation-stage depth criterion only;
it never proves source evidence or semantic completeness.
