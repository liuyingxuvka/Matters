# Matters Model Depth Maintenance

Read `references/service-contract.md`. Submit every assessment and missing-work
request through MatterService.

1. Bind the assessment to one occurrence, inventory revision, policy,
   provider, prompt/schema, active skill, and model identity.
2. Check terminal coverage, current extraction, terminal analysis, anchored
   evidence, original-owner terminal disposition, localization,
   representative visual, projection, and UI reachability where required by
   policy.
3. Return `sufficient` only when every required criterion is satisfied or explicitly non-applicable and no blocker remains.
4. Return `partial` when useful current analysis exists but criteria remain.
5. Return `blocked` for an unavailable required source, extractor,
   ResearchGuard operation, original-owner dispatch, localization, visual, or
   projection stage.
6. Return `stale` whenever a declared dependency identity changes.
7. Queue the exact missing owner work without editing product code,
   requirements, FlowGuard models, or another owner's state.

Ordinary uncertainty is allowed when the responsible owner has recorded a
terminal uncertainty-preserved disposition. It is not a human-review
requirement.
