# MatterService generated-hero contract

## Input

The private brief binds `matter_id`, semantic identity, hierarchy and merge
freshness, permission/safety disposition, generation-policy revision, exact
English/Chinese locale registry, bounded `topic_concepts`,
`theme_concepts`, style, negative constraints, and one `brief_fingerprint`.

The brief contains no source excerpt, literal text request, logo/brand, private
identifier, local path, email/message content, or identifiable real person.
The current policy requires photorealistic candid documentary/editorial
photography, natural available light, a plausible real-world environment, an
ordinary camera perspective, and only fictional generic non-identifiable
people when people are useful. The place, physical objects, equipment, and
activity must be Matter-specific enough to recognize the topic without a
caption. At least two independently recognizable Matter-specific physical
cues must dominate the frame; a person, pose, or generic work activity cannot
be the only distinguishing cue. A generic person at a computer or
interchangeable office scene is invalid. It explicitly forbids abstract/conceptual
illustration, vector/icon art, 3D/isometric rendering, infographic, collage,
diagram, poster, surreal/metaphorical composition, screenshots, document
previews, legible screen content, text, and logos.

## Output

Return browser-safe JPEG, PNG, or WebP bytes, exact `brief_fingerprint`,
runner-contract and execution identities, allowed safety disposition, and
equivalent non-empty English/Chinese alt text.
Alt text describes the photographic scene and cannot describe it as abstract,
conceptual, an illustration, a render, or a metaphor.

A current root Hero that fails the automated or browser visual-specificity
review is retired through the non-blocking `quality` invalidation. The owner
returns the Matter to one pending placeholder, prepares one replacement brief,
and requires the replacement to pass the same safety, privacy, root-only, and
bilingual-alt gates before publication.

The C11 owner validates bytes, privacy/safety and fingerprints, stores the
asset only in the private blob store, and issues a random opaque token. C12
projects only `generated_current`, `generation_pending_placeholder`, or
`generation_blocked_placeholder`, token when current, bilingual alt, and
generation revision.

## Boundaries

The hero is presentation-only and cannot become SourceVersion, EvidenceAnchor,
Matter identity, event proof, or an Images-gallery item. WorkItems and Events
never receive heroes. Only root Matters are eligible. Child Matters, Sources,
SourceVersions, and quick views are `not_applicable` and receive no generation
request, asset, or placeholder. Standard/compact root cards consume the same
image. Identity, topic, theme, merge, split, reparent, permission, safety, and
policy changes invalidate; ordinary clues, wording, translation, scans,
retries, receipts, and start-time changes do not. A generated photograph is a
presentation-only reconstruction and never asserts that the depicted scene
happened.

The route is model-independent and Codex-hosted. It never requests an API key,
calls a provider API directly, or uses a direct API fallback.

Shared handoff vocabulary: `deterministic_hard_exclusion`,
`deterministic_preprocessor`, `low_cost_annotator`, `ambiguity_resolver`,
`matter_modeler`, `hero_image_generator`, `consistency_reviewer`,
`maintenance_orchestrator`, `source_neighborhood_id`, `source_group_chain`,
`source_group_labels`, `source_spatial_context_revision`, parent, child,
WorkItem hierarchy audit, English/Chinese title, summary, topic type,
`generated_hero`, and `research_operation`. Canonical state never exposes an
absolute path publicly and never binds a named model, API key, direct API, or
fallback.

This presentation skill cannot create a Matter. Program, cache, and
internal-application records remain hard-excluded before this route.

Contract applicability: storage_pointer=not_applicable_no_source_original_access;
source_group=not_applicable_opaque_gate_only;
situation_graph=not_applicable_no_graph_node_access;
world_model=not_applicable_no_inference_access;
hero=applicable_root_only_generation_child_not_applicable;
unattended=applicable_bounded_generation_no_final_verification.

The route cannot read or copy source originals. It receives no SourceGroup
membership, SituationGraph node, or World Model inference and consumes only
the minimized current root-Matter brief. Child Matters are explicitly
`not_applicable`; they receive no image, placeholder, or generation request.
Missing machine installation selects this immutable bundled consumer for
internal use only and never authorizes a global installation.
